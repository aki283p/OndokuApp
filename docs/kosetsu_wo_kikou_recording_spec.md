# 「小説を聴こう」画面録画から読み取れた主な仕様

このドキュメントは、参考アプリ「小説を聴こう」の画面録画をもとに、画面遷移・UI・機能から読み取れた仕様を整理したものです。

OndokuAppへ機能追加する際の参考資料として使用します。

## 1. アプリ全体の位置づけ

「小説を聴こう」は、単なるURL読み上げアプリではなく、Web小説サイトをアプリ内で探し、作品単位・話数単位で保存し、連続再生する **本棚 + Webブラウザ + ダウンロード + 読み上げプレイヤー** 型のアプリです。

主な価値は以下です。

- Web小説を目で読まずに聴ける
- 複数話の小説を作品単位で管理できる
- 保存済みの話を連続して読み上げられる
- 読み間違いをユーザー辞書で補正できる
- 読書履歴・進捗を本棚で管理できる

## 2. 本棚画面の仕様

画面録画から、本棚画面には以下の要素が確認できました。

### 表示項目

- 登録済み小説の一覧
- 作品タイトル
- 作者名
- 取得元サイト名
  - カクヨム
  - 小説家になろう / 小説を読もう
  - アルファポリス
  - など
- 話数または取得済み話数の進捗
  - 例: `15/20`
  - 例: `141/141`
  - 例: `319/319`
- 星評価らしき表示
- ダウンロード状態
- 更新状態

### 操作

- 新しい小説を追加
- 小説を削除
- 小説を更新
- 小説を聴く
- その他メニュー
- 検索
- メニュー表示

### OndokuAppとの差分

現在のOndokuAppにも本棚画面はありますが、以下が不足しています。

- 作品単位と話数単位の分離
- 取得済み話数の表示
- 話数進捗の表示
- 作品更新機能
- 複数話ダウンロード状態の表示
- 作者名表示
- 取得元サイト名の詳細表示

## 3. Web小説追加フローの仕様

録画では、「新しい小説を追加」から、対応サイト一覧またはアプリ内ブラウザへ遷移していました。

### 対応サイト一覧

画面録画で確認できたサイト例です。

- 青空文庫
- 小説を読もう
- カクヨム
- アルファポリス
- ハーメルン
- ミッドナイト
- ノクターン
- ムーンライト

### アプリ内ブラウザ

録画では、アプリ内でカクヨムを開き、以下のような操作を行っていました。

- サイト内検索
- ランキング閲覧
- 詳細検索
- 作品ページへの移動
- 作品詳細画面の確認

作品ページでは、以下の要素が確認できました。

- 作品タイトル
- 作者名
- 作品説明
- フォロー
- 作品情報
- 話数
- 文字数
- ジャンル情報
- `この作品を追加する`
- `1話目から読む`

### OndokuAppとの差分

現在のOndokuAppは、URL入力欄へURLを貼り付けて1ページの本文を取得する方式です。

不足しているものは以下です。

- アプリ内ブラウザ
- 対応サイト一覧
- 現在開いている作品ページを追加する導線
- 作品ページから話一覧を解析する機能
- 作品単位での取り込み
- 複数話の一括取得

## 4. ダウンロード・更新機能の仕様

録画では、作品追加時に以下のような流れが確認できました。

- 小説をダウンロード
- 複数ストーリーを取得
- `20のストーリーをダウンロードしました` のような完了表示
- 一度にダウンロードできる話数制限
- 続きを読む場合は更新操作を促すメッセージ

### 想定仕様

- 作品には複数の話が存在する
- 初回追加時に一定数の話をまとめて取得する
- 取得数には上限がある
- 続きは本棚の更新操作で取得する
- ダウンロード完了後にダイアログを表示する

### OndokuAppとの差分

現在のOndokuAppは、`1つのNovel = 1つの本文` の構造です。

録画のアプリに近づけるには、以下が必要です。

- `Novel` と `Episode` の分離
- 複数話保存
- 取得済み話数管理
- 更新処理
- ダウンロード進捗表示
- 取得完了ダイアログ

## 5. 読み上げ画面の仕様

録画では、読み上げ画面に以下の要素が確認できました。

### 表示項目

- 作品タイトル
- 作者名

- 現在の話数

### 操作

### OndokuAppとの差分

OndokuAppには読み上げ画面、進捗表示、ハイライト、続きから再生があります。

不足しているものは以下です。

- 話単位の前後移動
- 作品内の話数表示
- 話タイトル表示
- 作者名表示
- 巻き戻し / 早送り操作
- 読み上げ画面から辞書登録する導線
- 話単位の進捗管理

- 停止時刻読み上げなどの細かい設定

## 8. OndokuAppに足りない機能一覧

### 優先度: 高

| 機能 | 理由 |
|---|---|
| 作品・話数モデル | 現在は1作品1本文。複数話管理に必要 |
| 複数話ダウンロード | Web小説アプリとして重要 |
| 更新機能 | 続きの話を取得するために必要 |
| 話単位の読み上げ画面 | 前話・次話・話数表示に必要 |
| アプリ内ブラウザ | URL貼り付けだけより使いやすい |

### 優先度: 中

| 機能 | 理由 |
|---|---|
| 対応サイト一覧 | ユーザーがサイトを選んで探せる |
| 読み上げ画面から辞書登録 | 読み間違いをその場で直せる |
| TTS音声選択 | 速度・ピッチ以外の調整ができる |
| テスト再生 | 辞書・音声設定で便利 |
| 作品専用辞書 | 作品固有の造語に対応できる |

### 優先度: 低

| 機能 | 理由 |
|---|---|
| 効果音設定 | 便利だが必須ではない |
| リモコン設定 | 後回しでよい |
| 星評価表示 | 取得元サイト依存のため後回し |
| 広告表示 | OndokuAppでは不要 |

## 9. 次に実装すべき順番

### 第1優先: 作品と話数の構造化

現在の `Novel` を拡張しすぎるより、将来的には以下のように分離するのが自然です。

```text
Novel
- id
- title
- author
- sourceUrl
- sourceSite
- isFavorite
- lastReadEpisodeId
- createdAt
- updatedAt

Episode
- id
- novelId
- title
- content
- episodeUrl
- episodeIndex
- currentPosition
- isDownloaded
- createdAt
- updatedAt
```

この構造がないと、前話・次話・更新・複数話ダウンロードが作りにくくなります。

### 第2優先: 複数話ダウンロード

まずは簡易版で十分です。

- 作品ページURLを入力
- 作品ページから話リンク一覧を抽出
- 最大20話まで取得
- 本棚に保存
- ダウンロード中表示
- 完了ダイアログ

### 第3優先: 読み上げ画面の話移動

- 作品タイトル
- 作者名
- 話タイトル
- `1/30` のような話数表示
- 前の話
- 次の話
- 現在話の読み上げ

### 第4優先: 読み上げ画面から辞書登録

- 語句を入力
- 読みを入力
- テスト再生
- 登録
